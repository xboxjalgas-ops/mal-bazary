# Мал Базары — фермерлерге арналған онлайн-нарық

Сиыр, қой, жылқы, тауық, қаз, үйрек, қоян сияқты малдарды сатуға/сатып алуға,
сондай-ақ мал шаруашылығына қажетті өнімдерді (жем, дәрі-дәрмек, құрал-жабдық)
іздеуге арналған веб-платформаның frontend прототипі.

## Құрылым

```
mal-bazary/
├── index.html                     — негізгі бет
├── market.html                    — index.html-мен бірдей (болашақ landing үшін)
├── css/style.css                  — дизайн, түстер, орналасу
├── js/app.js                      — тіркелу, хабарландыру беру, фильтр, қоңырау логикасы
├── netlify/functions/send-code.js — SMS-код жіберу (Mobizon арқылы, серверде)
├── netlify/functions/verify-code.js — SMS-кодты тексеру (серверде)
├── netlify.toml                   — Netlify баптаулары
└── README.md                      — осы файл
```

## Іске қосу

Backend қажет емес — таза HTML/CSS/JS.

1. Файлдарды компьютерге жүктеп алыңыз (немесе GitHub-тан clone жасаңыз).
2. `index.html` файлын кез келген браузерде ашыңыз.

**GitHub Pages арқылы жариялау:**
1. Осы `mal-bazary` қалтасын GitHub репозиторийіне жүктеңіз.
2. Repository → Settings → Pages бөліміне өтіңіз.
3. Source ретінде `main` branch және `/ (root)` таңдаңыз.
4. Бірнеше минуттан кейін сайт `https://<username>.github.io/<repo>/` мекенжайында ашылады.

## Қалай жұмыс істейді

- **Тіркелу**: телефон нөмірі (+7 форматында, автоматты бос орындармен) →
  SMS-код (демо режимде код әрдайым **1234**) → аты-жөні. Тіркелген соң ғана
  хабарландыру қосуға болады.
- **Хабарландыру беру**: тіркелген кез келген қолданушы (сатушы да, сатып
  алушы да) өзінің малын немесе қажетті өнімін/құрал-жабдығын өзі қосады —
  админ керек емес.
- **Байланыс**: сатыпалушы мен сатушы тек қоңырау арқылы байланысады, ішкі
  хабарласу (чат) жоқ — бұл алаяқтықтың алдын алуға көмектеседі.

## Тіркелу растауы — Telegram bot арқылы (тегін, әрі белсенді)

Қазір сайт тіркелуді SMS-тің орнына **Telegram bot** арқылы растайды — толығымен
тегін және лимитсіз. Қалай қосу:

### 1. Bot жасау (BotFather арқылы)
1. Telegram-да **@BotFather** деген ресми бот-пен сөйлесуді бастаңыз.
2. `/newbot` командасын жіберіңіз.
3. Bot-қа атау беріңіз (мыс. "Мал Базары Тіркелу").
4. Bot-тың **username**-ін беріңіз — ол міндетті түрде `bot`-пен аяқталуы керек
   (мыс. `MalBazaryRegBot`).
5. BotFather сізге **токен** береді (мыс. `123456:ABC-DEF1234...`) — оны сақтап
   қойыңыз, ешкімге бермеңіз.

### 2. Токенді Netlify-ге қосу
1. Netlify → **Site configuration → Environment variables → Add a variable**.
2. `TELEGRAM_BOT_TOKEN` = BotFather берген токен.
3. **Deploys → Trigger deploy → Deploy site** басыңыз.

### 3. Bot username-ін кодқа жазу
`js/app.js` файлында мына жолды тап:
```js
const TELEGRAM_BOT_USERNAME = "MalBazaryBot";
```
Оны өз bot-ыңыздың нақты username-іне ауыстырып, GitHub-қа қайта жүктеңіз
(Upload files → осы файлды ауыстыру).

### 4. Webhook-ті бір рет қосу
Сайт Netlify-де жарияланған соң, төмендегі сілтемені (өз токеніңіз бен сайт
адресіңізбен) браузерде **бір рет** ашыңыз:
```
https://api.telegram.org/bot<ТОКЕНІҢІЗ>/setWebhook?url=https://<сайтыңыз>.netlify.app/.netlify/functions/telegram-webhook
```
Жауапта `"ok":true` көрсе — дайын. Бұл — код жазу емес, тек браузерге сілтеме
теру ғана.

Содан кейін сайттағы "Telegram арқылы бастау" батырмасы толық жұмыс істейді:
қолданушы Telegram-ды ашып, нөмірін бір батырмамен бөліседі, сайт автоматты
түрде танып, тіркелуді аяқтайды.

*(Ескерту: `netlify/functions/send-code.js` және `verify-code.js` файлдары —
Mobizon арқылы нағыз SMS жіберудің баламалы нұсқасы, қаласаңыз кейін соған
ауыса аласыз. Қазір олар frontend-пен байланыстырылмаған.)*

## ⚠️ Маңызды: бұл — демо, өндірістік (production) сайт емес

Бұл нұсқада деректер тек браузердің жадында (JavaScript айнымалыларында)
сақталады және бетті жаңартқанда жоғалады. Нақты, көпшілікке ашық сайт жасау
үшін төмендегі қауіпсіздік талаптары **міндетті түрде** орындалуы керек:

### Backend және дерекқор
- Барлық хабарландырулар, пайдаланушылар мен телефон нөмірлері **серверлік
  дерекқорда** (мыс. PostgreSQL) сақталуы керек, клиент жағында емес.
- SMS-код жіберу нағыз SMS-провайдер арқылы (мыс. Mobizon, SMS.RU,
  Twilio) серверде іске асырылуы керек — API кілттерін ешқашан HTML/JS
  файлдарына немесе GitHub репозиторийіне қоспаңыз.

### Хакерлерден қорғау (production үшін)
- **HTTPS міндетті** — деректер шифрланбаған желі арқылы жіберілмеуі керек.
- **Серверлік валидация**: клиенттен келген деректерге (аты, телефон, баға,
  сипаттама) сенбеу керек — барлығын серверде қайта тексеру қажет.
- **XSS-тен қорғау**: пайдаланушы жазған мәтінді HTML ретінде тікелей
  көрсетпей, арнайы экрандау (escaping) қолдану керек — бұл прототипте
  іске асырылған (`escapeHTML` функциясы), бірақ нағыз сервер жағында да
  қайталануы тиіс.
- **SMS-код лимиті (rate limiting)**: бір нөмірге минутына/сағатына жіберілетін
  SMS санын шектеу — спам мен ботқа қарсы.
- **Құпия деректерді репозиторийге қоспау**: API кілттер, дерекқор
  парольдері `.env` файлында сақталып, `.gitignore`-ге қосылуы керек.
- **CAPTCHA немесе осыған ұқсас механизм** тіркелу мен хабарландыру қосу
  формаларында — автоматты боттардан қорғау үшін.
- **Модерация**: жаңа хабарландыруларды автоматты/қолмен тексеру жүйесі —
  алаяқтық хабарландыруларды алдын ала анықтау үшін.

Бұл тізім — толық емес, тек негізгі бағыттарды көрсетеді. Нақты сервис
құру кезінде қауіпсіздік мамандарымен (немесе тәжірибелі backend
әзірлеушімен) ақылдасқан жөн.
